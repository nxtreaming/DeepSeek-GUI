# Kun v0.2.27

这一版是 v0.2.26 之后的一次创作工作流与平台能力更新。重点是推出本地优先的 Kun Video Editor 和可与主 Agent 协同编辑的 Kun PPT，给 Design 增加 Motion 时间线与内联组件原型，同时重做 Code 右侧工作区、运行中引导和 Write 文档体验，并为跨设备数据迁移、Extension 媒体能力以及 Runtime 稳定性补齐基础设施。

### Kun Video Editor

- 新增默认随产品分发的 Kun Video Editor 扩展，在 Code 右侧工作区以独立标签打开；界面针对 280–760 px 的侧栏宽度重新设计，并适配中英文、主题、键盘和无障碍状态。
- 支持通过受保护的系统选择器导入本地媒体，建立版本化项目、素材、序列和时间线；原始文件路径不会暴露给扩展页面，编辑过程保持源文件不变。
- 新增字幕与转录优先的剪辑流程，包括 SRT/VTT/JSON 导入、单词时间戳、静音与口头语处理、字幕构建、波形、胶片条、素材重连和恢复。
- 时间线支持拖动、裁切、分割、缩放、吸附、波纹/覆盖编辑、音视频联动、淡入淡出、嵌套序列、关键帧和音视频效果；人工操作与主 Kun Agent 使用同一套带 revision 检查的结构化命令。
- 增加语音片段与画面时刻搜索、VAD、说话人、节拍与同步分析、多机位、代理文件、proof/preview、OTIO、项目包和专业导出等能力。
- 长时间分析、预览和导出任务支持持久化进度、取消、重启恢复、缓存配额和 revision 固定，生成或导出的媒体也会作为可预览、打开和定位的正式产物回到会话。
- 本版仍坚持本地优先：FFmpeg/ffprobe 不包含在 `.kunx` 中，需要由用户在系统中单独安装；涉及云端分析或生成的流程会先展示 Provider、上传边界和成本信息，再由用户确认。

### Kun PPT 与 Presentation Studio

- 新增默认随产品分发的 Kun PPT 扩展，主 Kun Agent 和用户可以在同一份演示稿上反复编辑，不需要进入第二个聊天或启动另一套 Agent Runtime。
- 每份演示稿保存为工作区根目录下的 `.kun-ppt.html`，内嵌版本化结构模型，并使用稳定 ID、revision 检查、幂等记录和串行写入保证人工编辑与 Agent 编辑的一致性。
- 右侧编辑器提供幻灯片缩略图、16:9 画布、文本/形状/图片、拖动缩放、层级与排序、撤销重做、预览和自动保存。
- 属性面板新增 DOM/Layers 树和受约束的 CSS 声明编辑，可调整布局、旋转、透明度、字体、边框、填充与图片适配；脚本、远程资源和任意 HTML 不会在带扩展桥接能力的页面中执行。
- Agent 完成演示稿写入后，会在最终回复下显示去重后的文件卡片，支持用系统默认应用打开或在文件管理器中定位。
- Kun PPT 面向可持续编辑的 HTML 演示稿；原生 `.pptx` 生成仍由 PPT Master 负责，两条流程互不替代。

### Design Motion 与内联组件原型

- Design 新增 Motion 创作模式，把时间线、属性轨道、类型化关键帧、缓动、循环方式、预设、自动关键帧、播放/暂停/重置和确定性拖动预览集成到现有画布。
- Motion 可以作用于原生画布对象以及 HTML、SVG 和运行中应用容器；带 SMIL/CSS 动画的 SVG 也会显示独立的内部播放状态，避免和外层可编辑轨道混淆。
- 动效数据以版本化 JSON 持久化，并接入撤销重做、快照、handoff、reduced-motion 和 Kun 结构化工具，让人工调整与 Agent 修改共享同一份模型。
- 新增 `design_component` 工具和 `component-designer` 子代理配置，可以直接把安全加固后的 HTML 组件原型发布到会话中，也可以在需要时委托专门的设计子代理生成。
- 内联组件卡片支持桌面/移动视口、采用、继续迭代、查看代码、复制和刷新；产物持久化到工作区，并通过离线 CSP、路径约束和大小/结构校验隔离运行。
- 白板 PNG 导出、生成图片回放和导出后预览进一步加固，`.kun-canvas` 本地产物目录也会默认加入 Git 忽略规则。

### Code 工作区、Chat 与 Agent 交互

- Code 保留右侧竖向工具栏，同时新增可关闭、切换和重排的右侧标签工作区；Files、Preview、Browser、Todo、Plan、Changes、Canvas、Subagents、Side Conversation 和扩展页面都可以保持各自状态，Terminal 继续作为独立底部抽屉。
- 标签顺序、当前选择、展开状态和宽度按工作区保存，并自动迁移旧的单面板状态；扩展仍使用自己的图标直接进入独立标签。
- 运行中的待发送文本新增“引导”操作，可以在下一个安全的模型边界注入当前回合；如果回合刚好结束或内容不支持引导，消息会继续保留在队列中，不会丢失。
- Agent 忙碌时可以提前为下一回合切换模型；Subagent 不再受固定时间上限限制，长时间委托任务更不容易被提前中断。
- Todo 进度新增悬浮详情，Reasoning rail、时间线跳转标记和消息动效重新设计，在长会话与窄窗口下更紧凑清晰。
- 会话可以直接展示交互式组件原型和生成媒体；图片导出后会自动进入预览，演示稿与媒体文件也使用统一的产物卡片操作。

### Write、文件与预览

- Write 新增专注模式、文档字数、在系统文件管理器中显示、编辑器显示偏好和首次使用引导，并补强工具栏与工作区文档操作。
- 发送给 Assistant 前会先确保对应文件线程存在并保存当前草稿；被拒绝或未成功发送的内容会恢复到 Composer，降低切换文件和并发保存时丢失文本的风险。
- Assistant 上下文按文件隔离，避免不同文档之间的编辑历史相互污染；设置响应、文件监听和草稿持久化的竞争条件也得到修复。
- 富文本编辑改进长代码块、列表标记、选区操作和补全词边界；PDF 文本选择会正确适配 UI 缩放，并在多行、跨页和窗口变化时保持更稳定的几何位置。
- 文件预览标签补齐打开、切换、关闭和恢复生命周期；文件树拖动可以直接生成 Composer 引用，不可用的自定义工作区目录也会提供更安全的 fallback。

### 跨设备数据迁移（灰度）

- 新增“设置 → 数据迁移”，可以把选定工作区、关联会话与事件、附件、Design/Write 产物和可迁移应用状态导出为版本化、带校验和的 `.kunpack`，并在 Windows、macOS 和 Linux 之间重新映射路径。
- 导出支持完整/精简预设、敏感内容检查和密码保护；API 密钥、OAuth、系统钥匙串、设备身份、信任与 permission grant、运行进程、缓存和日志永远不会进入迁移包。
- 导入会先执行不修改目标数据的完整性、兼容性、磁盘空间和路径安全检查，再让用户选择新目录以及保留两份、合并、备份后替换或跳过等冲突策略。
- 暂存、提交、回滚和恢复使用持久化事务记录；应用中断后可以继续或回滚，最终报告会列出路径映射、排除项、冲突决定和脱敏警告。
- 导入的工作区默认保持未信任，扩展、工作流、定时任务、Connect 渠道和外部动作不会自动启用。
- 该能力仍受发布开关控制：开发构建默认可用，打包构建只有在发行时设置 `KUN_DATA_MIGRATION_ENABLED=1` 才会开放新迁移；未开放的构建仍会保留中断任务的恢复入口。

### Extension 平台与默认扩展

- Extension 右侧页面改为和原生工具一致的直接入口，每个 `views.rightSidebar` 贡献都有自己的图标和隔离标签；Host 继续统一管理排序、尺寸、焦点、权限、会话和失败状态。
- 扩展启用确认、权限 grant 和 workspace 绑定进一步统一；用户显式禁用、卸载、回滚或选择开发版本后，默认扩展更新不会擅自覆盖这些选择。
- 扩展页面可以继承 Host 主题和背景层，管理中心会显示更完整的图标、权限与状态信息，窄侧栏中的授权对话框也支持滚动。
- Extension API v1.2 补充受保护媒体选择与 Range 读取、后台 Job、FFmpeg broker、生成产物、媒体分析与缓存等通用能力，并加强 workspace 隔离、资源配额、取消和进程清理。
- 新增默认 Social Media Sidebar，可在右侧标签中切换抖音、哔哩哔哩和小红书；远程页面使用独立持久化会话，并限制导航、弹窗、权限、下载和扩展桥接能力。
- UI Plugin 支持通过 CDP 注入 Host 主题和分层背景，同时隔离开发渲染缓存，降低多工作树或不同开发实例之间的资源串用。

### Runtime、安全与稳定性

- 修复 approval 在完成、取消、超时和 SDK 中断之间的竞争，过期授权卡会及时退出可操作状态，Chat 事件继续以 Runtime 记录为权威来源。
- MCP 工具 schema 在单个回合内保持冻结，避免动态刷新破坏模型工具历史；Skills 菜单会刷新目录，空工具重试达到边界后也会强制给出最终回复。
- 压缩历史不再重复投影，Replay 增加模型感知的比较策略，OTLP 只导出脱敏 span，并进一步收紧事件回放、遥测开关和诊断输出。
- 自动化凭据存储按环境与实例隔离，系统密钥读取会区分缺失、暂时不可用和真实失败；数据迁移 IPC 也只接受可信主工作台 frame 的调用。
- 修复 Windows 命令启动时的 `EPERM` 处理、主窗口异常恢复、嵌套 Git 仓库 Review、自定义工作区失效、开发环境缓存冲突和应用恢复路径。
- 发布前检查、默认扩展打包、原生媒体证据、跨平台冒烟和依赖版本得到更新，为 macOS、Windows 和 Linux 安装包补充更完整的门禁。

### 升级说明

- 从 `v0.2.26` 升级可直接通过 GUI 更新，现有工作区、Provider、MCP、Skills 和会话数据不需要手动迁移。
- Kun Video Editor、Kun PPT 和 Social Media Sidebar 通过默认扩展目录分发；显式禁用、卸载、回滚或开发版本选择会继续保留。
- 使用 Kun Video Editor 的本地探测、预览和导出前，请先安装可用的 FFmpeg/ffprobe。macOS 如需字幕滤镜，建议安装 `ffmpeg-full`；不同平台需要分别确认可用性。
- `.kun-ppt.html` 可以在 Kun PPT 中继续编辑，也可以用普通浏览器查看；需要原生 PowerPoint 文件时继续使用 PPT Master。
- 跨设备迁移尚处于灰度阶段，是否显示取决于安装包的发行开关；迁移前请保留重要数据的独立备份，并妥善保存迁移包密码。
- Social Media Sidebar 中的第三方站点登录状态保存在隔离会话中；使用时仍需遵守相应平台的账号、内容和网络规则。

### 总结

v0.2.27 让 Kun 的创作能力从“生成一个结果”继续走向“在同一个工作区持续编辑”：视频、演示稿、动效和组件原型都有了可检查、可继续修改的原生工作面，主 Agent 也能和用户共享同一份结构化状态。与此同时，右侧标签工作区、运行中引导、Write 稳定性、跨设备迁移和 Extension 通用媒体能力把这些流程连接得更完整；Runtime 与发布门禁的加固，则为更长、更复杂的本地任务提供了更可靠的基础。
